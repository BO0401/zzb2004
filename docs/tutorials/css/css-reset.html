CSS Reset 与基础样式规范（现代实践）

理解 Reset 与 Normalize 的差异，采用“现代 Reset + 合理默认值”的组合，建立稳定的样式基线。

1. Reset vs Normalize

  • Reset：尽量清空浏览器默认样式，提供“无差异”的起点。
  • Normalize：在保留合理默认的同时，统一各浏览器差异。
  • 现代实践：采用“轻量 Reset + 语义化默认值”的折中方案。

2. 现代 Reset（推荐基线）

/* 盒模型与 margin 清零 */
*,*::before,*::after{ box-sizing:border-box }
*{ margin:0 }
/* 高度与文字渲染 */
html,body{ height:100% }
body{ line-height:1.5; -webkit-font-smoothing:antialiased }
/* 媒体元素 */
img,picture,video,canvas,svg{ display:block; max-width:100% }
/* 表单元素继承字体 */
input,button,textarea,select{ font:inherit }
/* 断词与容器隔离 */
p,h1,h2,h3,h4,h5,h6{ overflow-wrap:break-word }
#root,#__next{ isolation:isolate }

这套规则在保持简单的同时，解决了大多数项目起步时的兼容性问题。

3. 合理默认与可访问性

列表与链接

ul[role="list"],ol[role="list"]{ list-style:none; padding-left:0 }
a{ color:inherit; text-decoration:inherit }
:focus{ outline:2px solid var(--accent); outline-offset:2px }
:focus:not(:focus-visible){ outline:none }

列表默认去掉圆点需配合 role="list"，便于语义与可访问性。

图片与版式

img{ height:auto }
article p + p{ margin-top:1em }
small{ font-size:.875em; color:#94a3b8 }

在排版容器内定义合理的段落间距，而不是全局。

5. 实战建议

  • 将 Reset 作为项目最先加载的样式（如 base.css），并在其后引入组件样式。
  • 不要过度清空所有样式，保留语义元素的合理默认（如 strong/em）。
  • 为 :focus-visible 提供清晰的焦点样式，既可访问又不打扰鼠标用户。

6. 练习

  1. 基于本页的 Reset，建立你自己的 base.css，并扩展段落/标题/代码的基础排版。
  2. 为表单元素设计一致的聚焦样式，考虑颜色对比度与可访问性。
  3. 为文章内容区域（article）添加语义化的段落间距与图片样式。
